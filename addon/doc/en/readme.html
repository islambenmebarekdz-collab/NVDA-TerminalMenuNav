Terminal Menu Navigator

An NVDA add-on that makes interactive CLI menus in terminals usable with speech. When command-line tools (npm wizards, inquirer-style prompts, Claude Code menus) show options you navigate with the arrow keys, they repaint the whole list on every key press, so NVDA re-reads all the options and you never know where you are. This add-on detects that pattern and announces only the pointed option and its position (for example: "cherry, 3 of 5").

How it works

The add-on follows a "read the state, not the event" principle: new-text announcements are only a trigger. On each change it reads the visible terminal buffer and locates the menu block there (consecutive lines where exactly one starts with a pointer marker such as > or ❯). When a new menu appears, the question line above it is announced once as well.

Gestures (remappable from Input Gestures)

  • NVDA+Alt+L: toggle the filtering on or off instantly (your safety valve if detection ever misfires).
  • NVDA+Alt+K: repeat the currently selected option and its position on demand.

Settings

In NVDA settings, category "Terminal Menu Navigator": enable/disable the filtering, and enable/disable position announcements (3 of 5).

Scope

Works in real consoles (Command Prompt, PowerShell, Windows Terminal) and in the VS Code integrated terminal. The add-on never touches other speech: when no active menu is detected at the bottom of the buffer, speech passes through unchanged.

Known limitations in this version

  • Detection is heuristic by nature; if it ever misbehaves, toggle it off temporarily with NVDA+Alt+M.
  • Braille is not filtered (it shows the raw text).
  • Multi-select menus that glue the pointer to the checkbox glyph (such as "❯◉") may not be detected yet.
